title: nginx

浅谈nginx

什么是nginx？
Nginx是一款轻量级的Web 服务器/反向代理服务器及电子邮件（IMAP/POP3）代理服务器，并在一个BSD-like 协议下发行。
何为反向代理？
下面说说我的肤浅的理解。
先说说正向代理。比如我们用vpn时 我们访问百度还是直接输入http://www.baidu.com ，我们并不关心代理服务器的存在
而反响代理正好相反，我们访问天猫时其实并不是访问的真正的数据服务器，而是直接访问的代理服务器，再由代理服务器通过一定的算法帮我们去连接真实的数据服务器，这种情况下我们并不知道真实服务器的存在，而是直接访问的代理服务器。
nginx安装
下面以ubuntu为例
首先从nginx下载地址
此时我们下载的为tar.gz的文件
解归档和解压缩 tar -xzvf + 上诉文件名
cd nginx（解压缩后的文件）
会看到一个configure的脚本文件 执行该脚本 ./configure
此时我们可能会缺少某个包报错

apt-get install libpcre3 libpcre3-dev
apt-get install openssl 
apt-get install libssl-dev


  make install  //进行安装

#
此时我们的nginx会安装在

/usr/local/nginx 
//nginx启动
cd /usr/local/nginx/sbin
./nginx //启动nginx
./nginx -s stop //停止nginx
./nginx reload //重新加载

负载均衡配置
nginx负载均衡算法常见的5中算法

下面说收iphash算法的具体配置
进行配置文件 

vim  /usr/local/nginx/conf/nginx.conf 
//配额文件如下
#配置负载均衡服务器以及其权重
upstream mytest{
      server  localhost:4000 weight=1;
      server  localhost weight=1;
     }

    server {
        listen       9010;
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
            root   html;
            index  index.html index.htm;
            proxy_pass http://mytest;
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }

按上诉配置进行配置后重启nginx ,我们将会看到localhost:4000
localhost
将会被交替访问。
